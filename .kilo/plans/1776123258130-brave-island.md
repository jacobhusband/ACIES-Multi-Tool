# Local Project Manager Revamp Plan

## Summary
Revamp the Local Project Manager tool to support bidirectional sync (server ↔ local), restrict access to deliverable context only, implement timestamp-based recommendations, and add automatic backup creation.

## Requirements Analysis

### 1. Keep existing server→local copy functionality
- Existing `copy_project_locally` backend API in `main.py:8882` works correctly
- Copy tab in the dialog handles this direction

### 2. Add local→server copy functionality
- Need new backend API or extend existing to support reverse direction
- Current sync tab needs redesign for bidirectional flow

### 3. Remove from general tools menu, keep in deliverable menu
- **Current state**: Tool card exists in `index.html:470-486` (tools-grid)
- **Solution**: Remove tool card from HTML or add CSS class to hide from general tools view

### 4. Use saved file path to locate server project
- Already works via `launchContext.projectPath` passed from deliverable
- `getLaunchContextProjectRoot()` extracts the path correctly

### 5. Predictable local project save format
- **Clarification needed**: What naming convention should be used?
  - Option A: `{project_id} {project_name}` (e.g., "260429 BofA, 330 S. Santa Fe")
  - Option B: `{project_id}` only
  - Option C: User-defined with default to project name
  - Path example: `C:\Users\JacobH\Documents\Local Projects\{format}\`

### 6. Check for newer files and default to appropriate tab
- **Logic needed**:
  - Compare file timestamps between local and server
  - If local files newer → default to "Local→Server" tab with recommendations
  - If server files newer → default to "Server→Local" tab with recommendations
- **UI changes**:
  - Modify `activeTab` default based on comparison
  - Pre-populate recommendations list

### 7. Backup creation for each copy direction
- **Server Archive**: `{project_root}\Archive` (e.g., `M:\Gensler\BAC\2026\... Archive`)
- **Local Archive**: `C:\Users\JacobH\Documents\Local Projects\0 Archive`
- **Implementation**:
  - Before copy, create timestamped backup of destination folder
  - Use format: `{original_folder}_{YYYYMMDD_HHmmss}`

## Key Technical Details

### Existing Backup Pattern (reuse for Local Project Manager)
- Server backup already implemented in `backup_project_drawings` at `main.py:8624-8625`:
  - Creates `{project_root}\Archive\{timestamp}` folder
  - Example: `M:\Gensler\...\Archive\20260413_163000\`
- This pattern will be reused for Local Project Manager backups

### Local Archive Path
- Fixed path: `C:\Users\JacobH\Documents\Local Projects\0 Archive`
- Will need to create this folder if it doesn't exist

### File Timestamp Comparison
- Use `os.path.getmtime()` to get file modification times
- Compare files recursively between local and server folders
- Return categorized results: newer-local, newer-server, equal
